The 20 instruction tools are the public surface of the server. Each tool accepts a `{ request: string; context?: string; options?: object }` payload and internally orchestrates skills.

<Aside type="note" title="Model class glossary">
  Model class badges describe **task complexity**, not cost to users — the class tells the orchestrator what tier of model can handle this task well:

  | Badge | Meaning | Recommended models |
  |-------|---------|-------------------|
  | <Badge text="Zero-Cost" variant="success" /> | Solved by zero-cost models — drafts, triage, templates | [GPT-4.1](https://platform.openai.com/docs/models/gpt-4.1), [GPT-5 mini](https://platform.openai.com/docs/models/gpt-5-mini) |
  | <Badge text="Efficient" variant="caution" /> | Low-cost models — routing, aggregation, fast classification | [Claude Haiku 4.5](https://www.anthropic.com/claude/haiku), [GPT-5.4 mini](https://developers.openai.com/api/docs/models/gpt-5.4-mini) |
  | <Badge text="Advanced" variant="danger" /> | Full-capability models — synthesis, security, judgment | [Claude Sonnet 4.6](https://www.anthropic.com/claude/sonnet), [Claude Opus 4.6](https://www.anthropic.com/claude/opus), [GPT-5.4](https://developers.openai.com/api/docs/models/gpt-5.4), [GPT-5.3-Codex](https://developers.openai.com/api/docs/models/gpt-5.3-codex) |
  | <Badge text="Cross-Model" variant="note" /> | Independent cross-model review lane | [Gemini 3.1 Pro](https://ai.google.dev/gemini-api/docs/models), [Claude Opus 4.6](https://www.anthropic.com/claude/opus) |

  → [GitHub model comparison](https://docs.github.com/en/copilot/reference/ai-models/model-comparison) · [Artificial Analysis](https://artificialanalysis.ai)
</Aside>

## Input Schema

All tools share the same input schema:

```json
{
  "request": "string (required)",
  "context": "string (optional)",
  "options": "object (optional, tool-specific)"
}
```
